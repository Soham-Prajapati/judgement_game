# Frontend (Flutter)

## Kotlin files in this folder

The `android/` folder is generated by Flutter for Android platform hosting.

- `android/app/src/main/kotlin/.../MainActivity.kt` is expected and auto-generated.
- It does not mean the app UI is in Kotlin.
- App screens and logic are Dart files under `lib/`.

## Run locally

1. Install packages:

```bash
flutter pub get
```

2. Run on Android emulator (backend on localhost:8000):

```bash
flutter run \
	--dart-define=API_BASE_URL=http://10.0.2.2:8000 \
	--dart-define=WS_BASE_URL=ws://10.0.2.2:8000
```

VS Code run button flow:
- Open [frontend/lib/main.dart](frontend/lib/main.dart)
- Choose launch config: "Flutter Emulator (Local Backend)"
- Press Run/Debug

This config is stored in [frontend/.vscode/launch.json](frontend/.vscode/launch.json).

Notes:
- Use `10.0.2.2` for Android emulator to access host machine localhost.
- Keep backend running on port 8000.

## Run tests

```bash
flutter test
```

## Build APK for friend testing

Build directly (script handles Java detection):

```bash
./build_friend_apk.sh https://YOUR_PUBLIC_BACKEND_URL wss://YOUR_PUBLIC_BACKEND_URL
```

APK output:

```text
build/app/outputs/flutter-apk/app-release.apk
```

If the script says Java is missing, open Android Studio once and try again.

Current machine note:
- Flutter is configured to use JDK at `/opt/homebrew/opt/openjdk@17/...`.
- You can verify with `flutter doctor -v`.

## Important for internet multiplayer

- Use a publicly deployed backend URL, not localhost.
- Use `https` for API and `wss` for WebSocket.
- All players must run an APK built with the same backend URL.
